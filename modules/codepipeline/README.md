# CodePipeline Module

This module creates an AWS CodePipeline with three stages (Source, Build, Deploy) that automates the CI/CD process from source code to ECS deployment, integrated with CodeBuild for building Docker images.

## Resources Created

- `aws_codepipeline` - Multi-stage CI/CD pipeline
- `aws_s3_bucket` - Artifact store for pipeline stages
- `aws_s3_bucket_versioning` - Versioning for artifact bucket
- `aws_s3_bucket_server_side_encryption_configuration` - Encryption for artifacts (KMS or AES256)
- `aws_s3_bucket_public_access_block` - Block all public access to artifact bucket
- `aws_iam_role` - IAM service role for CodePipeline
- `aws_iam_role_policy` - IAM policy with required permissions

## Usage

```hcl
module "codepipeline" {
  source = "./modules/codepipeline"

  name = "myapp"

  # Source configuration
  source_provider         = "GitHub"
  repository_name         = "myorg/myapp"
  branch_name             = "main"
  codestar_connection_arn = "arn:aws:codestar-connections:ap-southeast-1:123456789:connection/xxx"

  # Integration with other modules
  codebuild_project_name = module.codebuild.project_name
  ecs_cluster_name       = module.ecs.cluster_name
  ecs_service_name       = module.ecs.service_name

  # Optional: KMS encryption for artifacts
  kms_key_arn = "arn:aws:kms:ap-southeast-1:123456789:key/xxx"

  tags = {
    Environment = "production"
    Project     = "myapp"
  }
}
```

## Prerequisites

Before using this module, you need to:

1. Create a CodeStar Connection to GitHub/GitLab manually via AWS Console
2. Approve the connection in AWS Console (cannot be automated)
3. Pass the connection ARN as `codestar_connection_arn`

## Security

- S3 artifact bucket blocks all public access
- S3 versioning enabled for artifact history
- Artifacts encrypted with KMS when `kms_key_arn` is provided, otherwise AES256
- IAM policy follows least privilege principle
- CodePipeline only has access to specific S3 bucket ARN, not all buckets

## imagedefinitions.json

The Deploy stage expects a file called `imagedefinitions.json` in the build output. This file must be generated by CodeBuild during the build stage. Example content:

```json
[
  {
    "name": "myapp",
    "imageUri": "123456789.dkr.ecr.ap-southeast-1.amazonaws.com/myapp:latest"
  }
]
```

Add this to your `buildspec.yml` post_build phase:

```yaml
post_build:
  commands:
    - printf '[{"name":"myapp","imageUri":"%s"}]' $IMAGE_URI > imagedefinitions.json
artifacts:
  files:
    - imagedefinitions.json
```

## Assumptions

- Using **CodeStar Connections** for GitHub/GitLab integration
- Pipeline triggers automatically on push to specified branch
- ECS deployment uses **rolling update** strategy via ECS provider
- Artifacts are stored in S3 with versioning enabled
